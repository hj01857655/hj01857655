# GitHub 页面更新设计

日期: 2026-03-12

## 背景

用户需要更新 GitHub 个人主页 README 与 GitHub Pages 页面，重点突出当前最火的项目，明确要求包含 star 约 1.1k 的仓库，并按星标排序展示。

## 目标

- README 与 GitHub Pages 同步展示 Top 6 项目。
- 展示顺序按 GitHub Stars 降序。
- 排除 fork 仓库，仅展示原创仓库。
- 对 1 星并列项采用最近更新时间作为并列排序参考。

## 非目标

- 不引入自动化抓取或定时更新脚本。
- 不重做整体 UI 风格，仅补全并更新项目展示区。

## 方案选型

- 方案 A: 直接按 Stars 选 Top 6 并写入页面。成本最低，满足当前需求。
- 方案 B: 增加自动更新流程，成本高，非本次范围。
- 方案 C: 人工策展，需额外输入，本次不采用。

本次采用方案 A。

## 项目选择规则

- 数据来源: GitHub 用户仓库列表（source only）
- 规则: Stars 降序排列；并列时按最近更新时间优先
- 结果: 6 个项目列表写入 README 与网站项目区块

## 页面改动范围

- README.md
  - 替换“Featured Projects”与后续学习项目列表为 Top 6 展示。
- personal-website/index.html
  - 增加 #projects 区块并填充 Top 6 卡片。
  - 头图文案强调主打项目（star 1.1k）。

## 风险与回滚

- 风险: Stars 动态变化导致内容过期。
- 回滚: 通过 git 还原到上一个提交。

## 验证

- 通过本地预览与 README 渲染检查内容展示是否完整。
